# Codex Task Protocol

1. Read `AGENTS.md`, the request, relevant roadmap, and current Git state.
2. State prohibited actions and scientific gates before experimental work.
3. Inspect existing patterns before editing.
4. Use test-first development for behavioral code.
5. Keep data, outputs, checkpoints, and credentials outside Git.
6. Run focused checks during implementation and the full required verification before completion.
7. Review `git diff`, tracked-file policy, and claim registry before committing.
8. Report actual evidence, skipped checks, claim status, and locked-test status.

Never turn a missing artifact, planned model, proxy, or failed experiment into a positive claim.
