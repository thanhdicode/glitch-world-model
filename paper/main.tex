\begin{abstract}
We study reproducible video game glitch detection using visual-change, feature-distance,
lightweight latent-dynamics, and learned reconstruction baselines. Pair-suspect grouped
evaluation reduces obvious leakage. A five-seed selected-pipeline study provides diagnostic
locked-test evidence, while a real CUDA Conv3D autoencoder run is reported on validation only.
The Conv3D result is negative (AUROC 0.403865) and is retained as engineering evidence rather
than a performance claim. Real LeWorldModel and JEPA integration remain future work.
\end{abstract}

\section{Introduction}
Gameplay glitches are temporally varied and expensive to label. This work asks whether
latent-surprise signals can support a reproducible binary clip-level detection pipeline without
overstating temporal localization or model capability.

\section{Related Work}
We position the study against gameplay glitch benchmarks, video anomaly detection, and
joint-embedding predictive architectures \cite{tempglitch,glitchbench,lewm,ijepa,vjepa}.

\section{Problem Formulation}
Given a gameplay clip, a scorer emits one anomaly value. Current public labels support binary
clip/video-level analysis, not verified temporal localization.

\section{Method}
Verified scorers are frame difference, feature distance, the \texttt{mini\_latent} proxy, and a
compact Conv3D video autoencoder. The proxy is not LeWM or JEPA. Scorers retain a common CSV
interface and train-dependent methods fit train-normal clips only.

\section{Experimental Setup}
We use pair-suspect grouped splits with zero observed cross-split groups. Phase 6D performs
validation selection before one locked-test score per seed. Phase 6E trains on train-normal and
scores validation only; its locked test remains untouched.

\section{Discussion}
The experiments support the reproducibility framework more strongly than the latent-surprise
performance hypothesis. The high Phase 6E recall accompanies low precision and weak ranking.

\section{Limitations}
The subset is small, pair IDs are heuristic, labels are binary, and prior subset inspection
limits locked-test interpretation. No state-of-the-art, temporal-localization, LeWM, JEPA,
SIGReg, or neural locked-test claim is made.

\section{Conclusion}
The repository provides a cautious, reproducible lab for testing gameplay glitch detectors.
The next phase is a LeWM integration audit, not a LeWM result.
